Analytics — Guida & checklist collegamenti

Quali fonti dati collegare per il centro analytics di Simracing.Fan, come, in che ordine. Da fare con calma.

Il principio. Non esiste un "pacchetto analytics" unico per Claude Code: la strada vera è un connettore per piattaforma (quasi tutti server MCP ufficiali o API). La buona notizia: GA4, Matomo e WooCommerce hanno già un MCP ufficiale/nativo — niente roba community sospetta dove conta. Esegue Max, non tu: ogni riga qui sotto è un lavoro mio. Una regola tua resta: ogni collegamento è un'installazione → audit prima, si parte dagli ufficiali.

1. Checklist rapida (priorità → valore)

Ordine pensato per "un pezzo alla volta, partendo dal valore". 1 = parti da qui · 2 = subito dopo · 3 = quando il commerce/adv conta · ∅ = nessun buon collegamento.

2. I collegamenti, uno per uno

📈 GA4 — MCP ufficiale Google parti da qui

Cos'è: server MCP ufficiale di Google (repo googleanalytics/google-analytics-mcp) che dà a Max accesso ai report e ai dati grezzi via Data API + Admin API. Ufficiale, gratis, basso rischio.

Come (lo faccio io): serve una credenziale Google Cloud (service account JSON) con accesso alla proprietà GA4. Poi:

claude mcp add analytics-ga4 --scope user \
  -e "GOOGLE_APPLICATION_CREDENTIALS=/percorso/credenziali.json" \
  -e "GOOGLE_PROJECT_ID=ID_PROGETTO" \
  -- pipx run analytics-mcp

Da te: autorizzare/creare il service account in Google Cloud (passaggio OAuth/console che richiede il tuo login) — te lo guido passo-passo quando partiamo. Doc: developers.google.com/analytics/devguides/MCP.

🔍 Search Console — API ufficiale Google parti da qui

Cos'è: dati di ricerca (query, click, impression, posizioni, indicizzazione). Coppia naturale con GA4: massimo valore SEO per i contenuti Simracing.Fan. Google non ha un MCP ufficiale, ma l'API è ufficiale e matura; esistono MCP community read-only ben fatti (es. acamolese/google-search-console-mcp — OAuth, sola lettura, MIT) da auditare prima.

Come: autenticazione OAuth con il tuo account Google che possiede la proprietà. Gate: il consenso OAuth lo dai tu una volta; poi gira Max.

🟣 Matomo — MCP ufficiale (plugin) subito dopo

Cos'è: Matomo ha rilasciato un MCP Server ufficiale (plugin matomo-org/plugin-McpServer). Seconda lente analytics, privacy-first, già attiva sul sito.

Come: in Matomo → Amministrazione → Plugin attivare McpServer, poi Sistema → Impostazioni generali → McpServer per abilitarlo. Poi lato Max:

claude mcp add --scope user --transport http matomo \
  'https://TUO-MATOMO/index.php?module=McpServer' \
  --header 'Authorization: Bearer IL_TUO_TOKEN'

Sicurezza: il token_auth di Matomo dà gli stessi permessi dell'utente e non è limitabile a sola lettura. Conviene creare un utente Matomo dedicato con accesso in view solo a Simracing.Fan e generare lì il token. Lo imposto io.

🛒 WooCommerce — REST API ufficiale subito dopo

Cos'è: ordini, prodotti, clienti, fatturato dal sito. WooCommerce ha anche un MCP nativo nel core, ma è in Developer Preview (può cambiare) → per ora uso la REST API ufficiale, stabile e collaudata.

Come: in WooCommerce → Impostazioni → Avanzate → REST API generare una chiave (consumer_key / consumer_secret) in sola lettura. Poi Max interroga l'endpoint /wp-json/wc/v3/.

Gate: chiave in read-only — Max legge i dati, non modifica ordini. Niente scritture sul negozio senza un tuo ok esplicito.

📣 Meta Pixel + Conversions API (server-side) quando fai adv

Attenzione: questo non è un MCP, è infrastruttura di tracking. Lo standard 2025-26 per uno store WooCommerce:

  • Via semplice (consigliata): plugin ufficiale "Facebook for WooCommerce" — Pixel + Conversions API (CAPI) integrate, configurazione guidata.
  • Via controllo totale: server-side GTM (es. ospitato su stape.io) o Conversions API Gateway di Meta — più potente, più da gestire. Solo se l'adv diventa serio.

Quando: ha senso accenderlo quando parti con campagne a pagamento, non prima.

🏷️ Google Merchant Center quando vendi

Cos'è: feed prodotti per Google Shopping / free listings. Come: plugin ufficiale "Google for WooCommerce" sincronizza il catalogo → Merchant Center (Content API). Gate: creare l'account Merchant e l'invio feed sono azioni tue; Max prepara/controlla.

🔥 Microsoft Clarity segnali, non integrazione

Verità scomoda: Clarity ha solo una Data Export API sottile (poche metriche aggregate, finestra di pochi giorni) — non esporta le registrazioni di sessione e non ha un MCP. Utile come segnale leggero gratuito, ma il valore vero (heatmap, replay) resta da guardare a mano nella sua dashboard.

🚫 Hotjar non collegabile

Nessuna API pubblica per heatmap/registrazioni, nessun MCP: programmaticamente è un vicolo cieco. Se vuoi quel tipo di dato "data-driven", Clarity batte Hotjar perché almeno un'API gratis ce l'ha. Hotjar tienilo solo come uso manuale dentro la sua dashboard.

3. Piano sequenziato (dal valore)

  1. GA4 MCP ufficiale — dati già attivi, ufficiale, gratis, sicuro. Il primo passo concreto (~15 min, lo faccio io appena mi dai il via in Google Cloud).
  2. Search Console API — gratis, Google, alto valore SEO. Coppia naturale con GA4.
  3. Matomo MCP — già attivo, seconda lente con utente dedicato in sola lettura.
  4. WooCommerce REST API — leggere ordini/fatturato, quando il commerce conta.
  5. Meta Pixel + CAPI (plugin FB ufficiale) — quando parti con adv a pagamento.
  6. Merchant Center (plugin Google for Woo) — quando serve il feed Shopping.
  7. Clarity — API gratis per segnali di comportamento (accettando che è sottile).
  8. Hotjar — salta, o uso manuale.

4. Cosa NON ti serve sapere / fare

  • Non c'è da scegliere un "pacco di skill": le skill sono prompt, non danno accesso ai tuoi dati. Quello che conta sono i collegamenti qui sopra.
  • Non installi tu niente: i comandi claude mcp add li lancio io. A te restano solo i consensi/login (Google Cloud, OAuth) che richiedono la tua identità, e le approvazioni sui gate.
  • Audit prima di ogni community connector (Search Console MCP, ecc.) — è la tua regola post-incidente, la rispetto.

Stato: guida pronta, nessun collegamento ancora attivato — è il "prima" che hai chiesto. Quando vuoi partire, dimmi solo «colleghiamo GA4» e ti guido passo-passo dal primo gate Google Cloud.
